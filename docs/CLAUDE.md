# docs/

Project documentation directory.

## Contents

- `qa-checklist.md` — Manual QA checklist used to validate releases.

## Conventions

- Plain Markdown only; rendered on GitHub.
- Update `qa-checklist.md` whenever a new user-facing flow is added (wizard step, bulk-import variant, test-console feature).
- Do not put generated artifacts (Playwright HTML reports, coverage) here — those live in `playwright-report/` and `test-results/` at the repo root.
